ATP Season Prediction App

An end-to-end, deployed ATP prediction platform that combines matchup modeling, tournament simulations, and season-long league workflows in one live product.

Story

Turning fandom into a production-style product

This project started from a practical question: how can season-long ATP predictions feel both statistically grounded and easy for real users to interact with every week?

I built it to connect with family and friends through something I am genuinely passionate about: tennis. The goal was to make prediction and discussion around ATP events fun, social, and informed by real modeling instead of guesswork.

The result is a complete product loop I built end to end, from data ingestion and modeling to runtime APIs, frontend flows, and deployment on Google Cloud for real-world use.

Technical Architecture + Deployment

From pipeline to production app

The stack uses FastAPI, PostgreSQL (Supabase-compatible), SQLAlchemy, and XGBoost. Match data is ingested and transformed into Elo and context tables, then served through runtime predictors.

Core features include head-to-head prediction APIs, tournament-odds endpoints, and Monte Carlo draw simulation for round advancement and title probabilities.

Training artifacts are generated offline and loaded in production runtime, so the app can keep inference fast while preserving a reproducible training flow.

The system is deployed on Google Cloud as a live web app with authenticated league workflows, so people can make picks, track outcomes, and stay engaged with the ATP season together.
