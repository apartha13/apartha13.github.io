Project 1

RAG and Context Engineering

A retrieval-augmented generation system built over RapidFire AI OSS docs under a strict 2,000-token context budget.

AI

This project started with a practical constraint: answer documentation questions accurately without exceeding a 2,000-token context window, and still provide traceable source lines.

To solve that, my team built a single-turn RAG pipeline that retrieves candidate chunks, reranks them, and generates grounded answers over the RapidFire AI OSS docs.

Under the hood, the system uses RapidFire AI and LangChain abstractions with a custom line-aware token chunker, FAISS similarity retrieval (k=25), BAAI/bge-large-en-v1.5 embeddings, and BAAI/bge-reranker-large with top_n=1 before final generation.

The main learning was how strongly context quality affects output quality: reducing noisy context improved precision and made answers more consistent in final experiments.
